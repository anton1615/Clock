# 軌跡規格書：Android UI 優化與自訂設定實作

#### 1. 概述 (Overview)
本軌跡旨在深度優化 Android 端的用戶體驗，包含通知欄的視覺與功能修正、新增本地設定頁面以支援自訂計時與顏色，以及針對長時間專注場景的螢幕常亮功能。同時，我們將更新 Android 的應用程式圖示。

#### 2. 功能需求 (Functional Requirements)

**Android 通知欄優化:**
*   **狀態同步**：通知欄需即時反應「播放/暫停」狀態、當前階段（工作/休息）以及是否已與伺服器同步（Sync Status）。
*   **實時倒數**：修正通知欄倒數功能，確保其與主引擎同步。
*   **視覺設計**：
    *   背景顏色：採用設定之強調色（Accent Color）的**暗色版本**。
    *   文字顏色：使用**亮色/淺色文字**以確保可讀性。
*   **鎖定畫面 (Lock Screen)**：設定通知為 \VISIBILITY_PUBLIC\，確保在未解鎖的鎖定畫面上也能直接看到倒數。

**Android 設定頁面實作:**
*   **獨立頁面**：新增全螢幕設定 Activity 或 Compose 頁面。
*   **計時設定**：可調整工作 (Work) 與休息 (Break) 的時長。設定將於**下一次階段切換時**生效。
*   **強調色設定**：
    *   可分別為工作與休息階段選擇強調色。
    *   通知欄的強調色應自動跟隨此設定（並轉換為暗色調）。
*   **音效選擇**：
    *   提供系統內建通知音效選單。
    *   包含「無音效 (Silent)」選項。

**螢幕管理:**
*   **螢幕常亮 (Keep Screen On)**：新增設定選項。開啟後，若 App 位於前台且計時器正在執行，將防止系統自動進入睡眠或鎖定。

**應用程式圖示更新:**
*   將 Android 專案的應用程式圖示更新為 \@app.png\（需確認檔案路徑與尺寸）。

#### 3. 驗收標準 (Acceptance Criteria)
*   [ ] 通知欄能正確顯示倒數、暫停狀態與同步狀態。
*   [ ] 鎖定畫面可見完整倒數通知。
*   [ ] 設定頁面可成功修改時長與顏色，並在下一次切換時生效。
*   [ ] 階段切換時，調用設定的音效（或不發聲）。
*   [ ] 螢幕常亮功能在勾選後於計時期間生效。
*   [ ] App 圖示已更新。

#### 4. 超出範圍 (Out of Scope)
*   不實作通知欄的直接控制按鈕（如暫停/跳過）。
*   不改動 PC 端的任何邏輯。
